# 📌 How to Showcase Your Projects

Your pinned repositories are your portfolio. Here is how to make each one shine.

---

## Step 1 — Choose What to Pin

You can pin up to **6 repositories**. Choose them carefully.

### ✅ Pin These

- Projects with a **working demo or deployed URL**
- Projects that show **problem-solving and creativity**
- Projects with **good documentation** (a proper README)
- Projects in **languages/frameworks you want to be hired for**
- Projects with **stars or forks** (social proof)
- Projects that are **complete or actively maintained**

### ❌ Do Not Pin These

- Tutorials you followed step by step
- Forked repos you never contributed to
- Projects with zero commits in the last 2 years
- Repos with no description or README
- Projects that are broken or half-finished

---

## Step 2 — Set Repository Descriptions

Every pinned repo must have a short description. This is what appears under the repo name on your profile.

**How to set it:**
1. Open the repository
2. Click the ⚙️ gear icon next to "About"
3. Write a clear 1-line description
4. Add relevant **topics/tags** (e.g., `react`, `python`, `open-source`)
5. Add your website URL if it has a live demo

**Examples:**

| ❌ No Description | ✅ Good Description |
|-------------------|-------------------|
| *(empty)* | "Real-time collaborative code editor built with React and WebSockets" |
| "my project" | "REST API for a food delivery app — Node.js, Express, MongoDB" |
| "test" | "Machine learning model to detect plant diseases from leaf images (95% accuracy)" |

## Step 3 — Write a Great README for Each Pinned Repo

Every pinned repo needs a proper README. Use this structure:

```markdown
# Project Name

> One sentence that describes what this project does and who it's for.

![Demo GIF or Screenshot](link-to-image)

## ✨ Features

- Feature 1
- Feature 2
- Feature 3

## 🛠️ Built With

![React](badge) ![Node.js](badge) ![MongoDB](badge)

## 🚀 Getting Started

### Prerequisites
- Node.js v18+
- MongoDB

### Installation

```bash
git clone https://github.com/username/repo-name
cd repo-name
npm install
npm start
```

## Step 4 — Add a Demo GIF (Game Changer)

A demo GIF in your README makes visitors understand your project in **5 seconds**.

**How to create one:**

1. Use **LICEcap** (Windows/Mac) or **Peek** (Linux) to record your screen
2. Keep it under 30 seconds
3. Upload it to your repo in an `/assets` folder
4. Reference it: `![Demo](./assets/demo.gif)`

> 💡 Projects with demo GIFs get **3x more stars** than those without.

---

## Step 5 — Add Topics to Your Repositories

Topics make your repo discoverable via GitHub search.

**How to add:**
1. Click ⚙️ gear next to "About" in your repo
2. Add 5-10 relevant topics
3. Use common terms: `python`, `react`, `machine-learning`, `open-source`, `beginner-friendly`

---

## ✅ Project Checklist

For each pinned repository:

- [ ] Has a clear, one-line description set
- [ ] Has relevant topics/tags added
- [ ] Has a proper README with at least: description, features, tech stack, how to run
- [ ] Has a screenshot or demo GIF
- [ ] Has a working live demo link (if applicable)
- [ ] Has a license file
- [ ] Is not broken — can be cloned and run
- [ ] Last commit is within a reasonable time frame
